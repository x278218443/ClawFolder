# 橘鸦Juya AI 早报 — 完整风格指南

## 一、视频格式

- **画面比例**：16:9 横屏（B站标准）
- **分辨率**：1920×1080 或更高
- **时长**：1.5-3 分钟（每条新闻 10-16 秒）
- **新闻条数**：每期 6-12 条

## 二、标题格式

- 格式：`[公司/产品] [动作] [功能]【AI 早报 YYYY-MM-DD】`
- 两条重磅新闻时用分号：`新闻A；新闻B【AI 早报 YYYY-MM-DD】`
- 标题只放最吸引眼球的那条新闻

## 三、描述区格式

每条新闻一行，格式：`MM:SS 公司/产品 动词 功能/事件`

示例：
```
00:09 豆包拟推三档订阅服务，月费最高五百元
00:24 Hermes Agent 升级 Hermes Kanban，Trinity 模型一周免费
00:38 TinyFish 宣布搜索与抓取功能免费面向开发者开放
```

## 四、视觉风格（从封面图分析）

### 封面图特征
- **拼贴式构图**：2-3 张相关图片拼在一起
- **左侧大图**：公司总部/产品界面/相关人物照片
- **右侧/下方**：产品 logo、界面截图
- **深色科技感背景**
- **标题文字叠加**：白色粗体大字，黑色描边
- **真实照片为主**：不是 AI 生成图，是真实的产品截图、公司照片、新闻配图

### 视频内画面特征
- 每条新闻独立画面
- 画面包含：产品 logo/截图、公司相关图片
- 底部字幕
- 快速切换（每条 10-16 秒）

### 字幕风格
- 底部居中白色字幕
- 可能有新闻标题在画面中上方

## 五、脚本/旁白风格

### 语言特征
- **新闻播报体**：客观、专业
- **中英文混用**：产品名/公司名用英文，描述用中文
- **句式简洁**：每句话只传递一个信息点
- **无废话**：没有"大家好"之类的开场

### 结构
- 第一条是标题故事，多说几句（背景、影响）
- 后面快速带过，每条 10-15 秒
- 简短过渡（"与此同时"、"另外"、"值得一提的是"）
- 结尾无总结

### 语气
- 像新闻联播主播，不是 YouTuber
- 不说"我觉得"、"我认为"
- 用"据报道"、"据悉"

## 七、我们需要做的关键改造

1. **画面比例**：竖屏 9:16 → 横屏 16:9
2. **脚本模式**：单话题 → 多话题拼接（6-12 条/期）
3. **配图来源**：随机图 → 新闻相关图（产品截图、公司照片）
4. **时长**：44 秒 → 1.5-3 分钟
5. **节奏**：每条新闻独立画面，快速切换
